```
[クリエイター]
    │
    ├─▶ Studio (studio.html)          ← 制作・編集
    │       │
    │       ▼
    │   Pressroom (pressroom.html)    ← レンダリング・公開

    │       │
    │       ▼

```

| 拡張子 | 説明 |
|---|---|
| `.dsp` | Studio プロジェクトファイル（R2 に保存） |
| `.dsf` | 公開済みレンダリング出力（R2 に保存） |

**URL 構造:**
```

                                                 ?page={n}&lang={code}
```

- `press_id` は暗号論的ランダム・永続（ISBN 相当）
- 公開可否は Pressroom のトグルのみで制御

## DSF レジストリ（将来拡張）

### コンセプト
DRM フリーでも「いつ・誰が・この作品を作ったか」を第三者が確認できる記録を残す。
クリエイターが著作物をスケールさせる際の管理台帳、および著作権侵害時の第三者エビデンスとして機能する。

### 文化庁著作権登録制度との連動
- 参照: https://www.bunka.go.jp/seisaku/chosakuken/seidokaisetsu/toroku_seido/
- 著作権は創作時点で自動発生（登録不要）
- 文化庁登録は「公表した事実」がある場合のみ申請可能
- **DSF レジストリが「公表の証拠」となり、文化庁登録の申請材料になりうる**

```
DSF レジストリ（民間・タイムスタンプ記録）
        │ 公表の証拠として
        ▼
文化庁 著作権登録（公的・第三者対抗要件）
```

### レジストリが持つフィールド

| フィールド | 用途 | 備考 |
|---|---|---|
| `press_id` | 作品の永続識別子 | 既存。変更不要 |
| `publisher_id` | 著者の識別子 | 既存 |
| `registered_at` | 登録日時（変更不可） | タイムスタンプ証明の核心 |
| `file_hash` | .dsf のハッシュ値 | ファイルの同一性・改ざん検知 |
| `published_at` | 公表日 | 文化庁登録申請に必要 |
| `published_url` | 公表場所の証跡 | canonical_url と同値 |
| `title` / `author` | 人間が読めるメタデータ | |

### 実装方針
- Pressroom Step 3 に「レジストリに登録する」トグルを1つ追加するだけ
- Firestore に `registry` コレクションを追加
- ポータル公開時に `published_at` / `published_url` を自動セット
- 非公開作品でも手動で登録申請できる（任意）
- `registered_at` は登録後に変更不可とする（証拠性の担保）
- 将来的に `registry_id`（人間が読める登録番号）を別途発番可能
  例: `DSF-2026-00042`（`press_id` は変更せず、別フィールドとして追加）

### 「公表」の定義と DSF 公開モードの関係

著作権法上の「公衆」＝「不特定の者」または「特定多数の者」（著作権法第2条5項）。

| 公開モード | アクセス可能な範囲 | 「公表」該当 |
|---|---|---|
| ポータル公開 | 誰でも（検索・一覧から発見可能） | ✅ 明確に該当 |
| 限定公開（URL のみ） | URL を知る不特定の誰でも | ✅ おそらく該当 |
| メール添付 .dsf | 特定の受取人のみ | ❌ 非該当の可能性 |

**設計上の含意：**
- 限定公開 URL を発行した時点で「公表した事実」が成立しうる
- ポータルへの一覧掲載は「公表」の必要条件ではない
- よって Pressroom で限定公開 URL を発行した瞬間が `published_at` の記録タイミングになる
- これにより「非公開作品でも文化庁登録申請の資格が生まれる可能性がある」

> ⚠️ 上記は設計上の解釈メモであり、法的確実性が必要な場面では著作権専門の弁護士に確認すること。

### 野良ファイルの扱い
- 未登録ファイルは完全に許容（フォーマットとして制限しない）
- ビューアーで `press_id` をレジストリ照会し「登録済み ✓」「未登録」を表示（将来）
- 登録を強制しない。あくまで任意のオプトイン
