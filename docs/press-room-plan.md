# Press Room Plan

更新日: 2026-03-01  
ステータス: Draft v0.1

## 1. 目的

Press Room は、作品を編集する `Studio` から分離された「公開運用」専用画面である。  
以下を明確に分離する。

1. Studio: 作品編集・保存
2. Press Room: 公開版管理・URL管理・公開ステータス管理

この分離により、将来の複数人運用（ロール管理・承認・課金管理）に拡張しやすくする。

## 2. 背景

現状は Studio で編集と公開操作を同時に行っているため、次の課題がある。

1. 公開運用の責務が混在し、操作意図が分かりづらい
2. 公開版の履歴や状態管理がしづらい
3. 将来の `publisher` ロール分離に対応しにくい

## 3. 対象範囲（MVP）

Press Room v1 の対象は次の通り。

1. 公開対象プロジェクトの一覧表示
2. 公開ステータス切替
3. 公開URLの発行・確認
4. 公開日時・更新日時・公開言語の表示
5. 配布元（Channel）情報の表示

MVPでは、承認ワークフローや予約公開は対象外とする。

### v1 運用制約（確定）

1. v1 は無料チャンネルのみを対象とする
2. 課金関連（請求書・領収書・未払い管理）は v2 以降
3. 投げ銭入金先など決済連携機能は v2 以降

## 4. 情報設計（画面）

Press Room では最低2画面を提供する。

1. `press.html`（公開版一覧）
2. `press-release.html?releaseId=...`（公開版詳細）

一覧で見る項目:

1. 作品タイトル
2. サブタイトル
3. Channel名（配布元）
4. 公開ステータス（draft/published/archived）
5. 公開URL
6. 最終更新日時
7. 公開言語

## 5. ステータスモデル

公開版ステータスは以下を基本とする。

1. `draft`: 下書き（非公開）
2. `published`: 公開中
3. `archived`: 公開停止（履歴保持）

遷移ルール:

1. draft -> published
2. published -> archived
3. archived -> published

## 6. URLポリシー（MVP）

MVPでは単純な公開URL方式を採用する。

1. URLは releaseId 単位で固定
2. `published` のみ一般閲覧可能
3. `archived` は閲覧不可
4. URL再発行は v2 で対応（token rotate）

## 7. ロール前提（MVP）

Press Room 操作の最小権限は次の通り。

1. `viewer`: 閲覧のみ
2. `editor`: 閲覧 + メタ更新（公開操作不可）
3. `publisher`: 公開/停止操作可
4. `admin`: publisher + メンバー管理（MVP外機能を将来追加）
5. `owner`: 請求/所有権管理（MVP外機能を将来追加）

## 8. データ依存（MVP）

MVPで必要な公開側データ:

1. タイトル/サブタイトル（多言語）
2. 著者（多言語）
3. Channel名
4. 公開言語
5. 公開ステータス
6. 公開日時・更新日時
7. サムネイル

補足:

1. 作品の「著者」と配布元「Channel」は別概念として保持する
2. Channel情報は project単位入力ではなく、チャンネルプロフィール由来とする

## 9. 主要ユースケース

1. Publisher が draft 作品を公開する
2. Publisher が公開中作品を archived に戻す
3. Viewer が公開URLとステータスを確認する
4. Editor がタイトル・サブタイトル・公開言語を更新する

## 10. 導入フェーズ

フェーズ1（MVP）:

1. `press.html` 新設
2. 一覧表示 + 公開切替
3. URL表示 + コピーボタン

フェーズ2:

1. release履歴（版管理）
2. 予約公開
3. URL再発行

フェーズ3:

1. 承認フロー（review -> approve -> publish）
2. 監査ログ
3. ロール別ダッシュボード

## 11. 非機能要件（MVP）

1. 公開切替は 3 秒以内に UI 反映
2. 一覧 100件までは 1 ページで操作可能
3. エラー時は再試行導線を必須化

## 12. 未決事項

1. release の採番ルール（手動/自動）
2. URL設計（projectベースかreleaseベースか）
3. archived 状態の保持期間
4. Press Room の初期ナビ位置（Portal直下 or Studio直下）
